  • Step 1: Auto Register

    • Auto Register to access the functionality of the app
    • To access the functionality provided by the Reference Application, you need to register using a email and password.
      To reduce the barrier to entry for new users, Auto Register functionality is implemented using a auto generated user stored in a browser cookie.
      If the user feels that the functionality of the application is suitable for short term engagement, then they will specify a User ID and password.
      As the User uses the application by creating content, they automatically have the incentive to safeguard the content they created which may be lost if the cookie in his browser is lost. This gives them an incentive to specify a User ID and Password. Also if the user wants to share the content created across browsers, then they will be forced to create a user id and password.
      Another advantage from a system maintainence point of view is that we can auto clear the temporary account after a certain period and this also eliminates users who are not our target audience.
      Auto Registration is controlled by creating a Auto Registration Invite specifying the number of auto registrations allowed in the Invite Module. The Invite Module is accessable only by users having AdminRole.

Step 2 : Enable Author Role

  • Enable Author Role to access editing functionality in the App. Auto Register may be configured to enable Author Role Automatically.
  • Security control for doing actions are controlled by predefined roles as well as the currently enabled default role defined in the User Details Screen.
    For enabling editing of InfoList, Author Action is required
