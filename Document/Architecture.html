Glossary & Notes

  • Earning based on ultimate value created by end user (Startups) is better than earning based on value created by your effort (Freelancer) is better than earning based on effort (Employee).
  • entrepreneur is one who can conceptualize an idea to be implemented with the minimum cost of resource and who can propagate it exponentially to capture the monetary value created as return on investment
  • Success in capturing value from an Idea involves right conceptualization/creation and right education/propagation
  • Capturing value from an idea using multiple people is better than going individually.
  • You can earn extraordinary return ethically only on other peoples effort, the trick is your small effort should have enabled other people to realize their effort so that they don't mind sharing a part of the value created.
  • Investor(s) reduce their risk by investing on many Startups where as Founder(s) increase their risk by concentrating on a single Startup.

Business Architecture Vision

  • The Architecture of this framework is based on Unconventional thought by a Professional Project Manager who had no coding experience
  • The Application has been Architecture based on the following vision
    • Focus on Iterative Conceptualization of an Idea
    • Focus on conceptualizers who cannot pay with money but can pay in kind with hard work
    • Focus on enabling outsourcing for individuals.
    • Focus on clear separation of skills/parallel execution/management of work by different actors involved in the conceptualization of an idea
    • Focus on non technical conceptualizers who can only may be learn HTML.
    • Focus on Platform and Language Agnostic Development Flow
    • Focus on write once, deploy anywhere vision, basically same source code deployed in PC, Servers, Cloud & Mobiles
    • Enable seamless change from web (Thin Client) based to native OPEN GL (Thick Client) based UI
    • Focus on doing things the easy/optimum way instead of the right/performant Way
    • Focus on RPC/REST API based development with minimum use of JavaScript on the client side
